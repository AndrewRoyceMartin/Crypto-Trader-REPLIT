Likely causes (in order of probability):

Bootstrap/legacy .table classes are being injected (often by app_legacy.js) only on the ML table → it falls back to Bootstrap skin.

Specificity mismatch: your v02 rules don’t beat older selectors like .table-responsive .table td (seen in legacy CSS).

Load order: v02 styles not last, so older CSS wins.

Different markup: ML table uses .table-v02--compact and also has .table/.table-hover stuck on it.

Below is a tight fix set—drop it in and the ML table will match the rest.

Plan

Enforce v02 styles with higher specificity (including .table-responsive …).

Neutralize any .table, .table-striped, .table-hover, etc., that JS might add.

Confirm load order in base_layout.html.

B) Runtime sanitizer (removes Bootstrap/legacy classes if JS adds them)

Put this after your main dashboard script (end of the template).

<script>
document.addEventListener('DOMContentLoaded', () => {
  document.querySelectorAll('.table-v02, .table-v02--compact').forEach(el => {
    // Remove Bootstrap/legacy classes that hijack styles
    ['table','table-striped','table-hover','table-sm','data-table','legacy-table']
      .forEach(c => el.classList.remove(c));
    el.setAttribute('data-style','v02');
  });
});
</script>

C) Load order (verify in base_layout.html)

Ensure v02 CSS comes last:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css">
<link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}?v={{ cache_version }}">
{% block extra_css %}{% endblock %}  <!-- v02 rules land here, after Bootstrap & legacy -->
